# AI 提示词：复刻"流水 TodoList"

## 提示词全文

---

```
我想让你帮我开发一个移动端优先的待办事项应用（TodoList），
设计风格为"流水禅意"——柔和、自然、宁静。
参考网站：https://liushui-todolist.netlify.app

## 一、整体风格

### 设计理念
- 风格关键词：禅意、自然、柔和、专注
- 不要过于花哨，保持克制和优雅
- 让用户在管理任务时感受到宁静

### 配色方案（必须严格遵循）
- 主背景色：#f8faf9（暖白带微绿）
- 次背景色：#eef4f1
- 卡片背景：#ffffff
- 主色调：#4a9079（薄荷绿）
- 辅助色：#7bc4a8（浅薄荷绿）
- 文字主色：#2d3a36
- 文字次色：#6b7d76
- 文字弱色：#a0b0a8

### 字体
- 标题字体：ZCOOL XiaoWei（站酷小薇体），Google Fonts CDN 引入
- 正文字体：Noto Serif SC（思源宋体），Google Fonts CDN 引入
- 中文字体必须支持中文显示

### 圆角系统
- 大圆角：24px（用于卡片、模态框）
- 中圆角：16px（用于任务项、按钮组）
- 小圆角：12px（用于输入框、小按钮）
- 胶囊：50px（用于筛选标签）

### 阴影
- 柔和阴影：0 4px 20px rgba(45, 58, 54, 0.08)
- 悬浮阴影：0 12px 40px rgba(45, 58, 54, 0.12)

## 二、页面结构

### 整体布局（从上到下）
1. 顶部 Header 区
   - 左上：小标题（13px）+ 主标题（32px，站酷小薇体）
   - 右上：当前日期（显示"星期几 月 日"，如"星期一 三月三十日"）
   - 下方：两个统计卡片并排
     - 卡片1：全部任务数量 + 图标
     - 卡片2：已完成数量 + 图标
   - 图标用 emoji，表情要温和（📋 ✓）

2. 筛选标签栏
   - 三个胶囊按钮：全部 / 待完成 / 已完成
   - 选中状态：背景变为 #4a9079，文字白色
   - 未选中：背景白色，带阴影

3. 任务列表区
   - 每个任务卡片包含：
     - 左侧：圆形复选框（未选中时灰色边框，已选中时绿色渐变+勾）
     - 中间：任务标题 + 任务描述（可选，小字）+ 优先级标签 + 创建时间
     - 右侧：删除按钮（🗑 emoji）
     - 左边缘：4px 宽的渐变色条（完成时有，未完成时隐藏）
   - 优先级标签颜色：
     - 高：#ffe5e5 背景 + #e63946 文字
     - 中：#fff3e0 背景 + #f57c00 文字
     - 低：#e8f5e9 背景 + #2e7d32 文字

4. 空状态
   - 大号 emoji（🌿）+ 标题 + 描述文字
   - 居中显示，柔和的浮动动画

5. 右下角 FAB（悬浮添加按钮）
   - 圆形，直径 60px
   - 绿色渐变背景（#4a9079 → #7bc4a8）
   - 白色 + 加号
   - 轻微阴影
   - 点击时旋转 45 度

6. 底部弹出模态框（添加任务表单）
   - 从屏幕底部滑入，带圆角
   - 顶部有一个灰色小把手（40px × 4px）
   - 表单内容：
     - 任务名称输入框（必填）
     - 任务描述文本框（可选）
     - 优先级选择：三个并排按钮，低/中/高，选中时对应颜色高亮
     - 取消按钮（灰色）+ 添加按钮（绿色渐变）
   - 点击遮罩层可以关闭

## 三、动画细节

### 背景动画
- 三个大圆圈（200-300px），用 blur(80px) 模糊
- 渐变填充，颜色取自配色方案
- 缓慢漂浮动效（20-25秒周期），无限循环

### 任务动画
- 进入：从下方 fadeIn + translateY，0.5秒，带 stagger 延迟
- 完成：scale 缩小 + 淡出，0.5秒
- 删除：向右滑出 + 淡出，0.4秒

### FAB 动画
- 页面加载：从 scale(0.5) rotate(-180deg) 弹出，0.5秒
- 点击：旋转 45 度

### 模态框动画
- 出现：从底部 translateY(100%) 滑入，0.4秒
- 消失：滑出

### Toast 提示
- 从顶部滑入，2.5秒后滑出

## 四、交互逻辑

### 数据结构
```javascript
{
  id: "唯一ID（用时间戳）",
  title: "任务名称",
  desc: "任务描述（可选）",
  priority: "high | medium | low",
  completed: false,
  createdAt: 时间戳,
  completedAt: null
}
```

### 核心功能
1. 添加任务 → 打开底部表单 → 填写 → 提交 → 保存到 localStorage → 渲染列表
2. 完成任务 → 点击复选框 → 更新 completed 状态 → 播放动画 → 更新统计
3. 删除任务 → 点击删除 → 确认提示 → 播放动画 → 从 localStorage 移除
4. 筛选任务 → 点击标签 → 显示/隐藏对应任务

### 数据持久化
- 使用 localStorage，key 为 "流水-tasks"
- 每次修改后立即保存
- 页面加载时读取并渲染

### 统计数字
- 全部任务 = tasks.length
- 已完成 = tasks.filter(t => t.completed).length

## 五、技术要求

### 响应式
- 移动端优先设计
- viewport 设置：width=device-width, initial-scale=1.0, user-scalable=no
- iPhone 刘海屏适配：使用 env(safe-area-inset-*)
- PC 端最大宽度 600px，居中显示

### 兼容性
- 纯 HTML + CSS + JavaScript，无任何框架依赖
- 单文件实现，所有代码写在一个 HTML 文件里
- 使用 CSS 变量统一管理主题色
- 动画优先使用 CSS，JS 只负责逻辑

### 性能
- 动画使用 transform 和 opacity，启用 GPU 加速
- 不要使用 js 动画库
- 图片使用 emoji 或 CSS 绘制，避免加载外部图片

## 六、输出要求

1. 给我完整的单文件 HTML 代码
2. 代码要有清晰的注释
3. 确保在手机浏览器中能正常打开和使用
4. 部署到 Netlify，给我一个可以访问的网址
```

## 使用说明

### 使用方式

1. 把上面的提示词完整复制给 AI（ChatGPT / Claude / Kimi 等）
2. 如果 AI 说"好的"之后开始写代码，让它继续
3. 如果 AI 写完后让你确认，告诉它"确认，开始写代码"
4. 写完后让它部署到 Netlify 或 Vercel

### 可能需要补充的追问

如果 AI 生成的版本有偏差，可以继续追问：

| 问题 | 追问 |
|------|------|
| 动画不够流畅 | "动画要使用 CSS @keyframes，而不是 JS" |
| 颜色不对 | "主色调是薄荷绿 #4a9079，不是蓝色或紫色" |
| 表单不弹出 | "模态框要从底部滑入，用 transform: translateY" |
| 手机点击无效 | "移除 touchend 的 preventDefault，避免阻止 click 事件" |
| 数据不保存 | "必须用 localStorage 持久化，key 为 '流水-tasks'" |
