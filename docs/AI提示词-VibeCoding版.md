# 流水 TodoList — AI 提示词（Vibe Coding 版）

> 用感受和直觉来描述你想要的东西，让 AI 自己决定技术方案。

---

## 第一步：画出一个你想要的页面

```
我想做一个待办事项应用，叫"流水"。

感觉是：安静的、自然的、像在溪边做事一样轻松。

配色方面：用绿色系，不要太亮也不要太暗，
像薄荷茶那种绿，让人看了不会紧张。

字体希望有点文艺感，但不是那种花哨的艺术字，
是那种让人看了觉得"这个设计师有品味"的感觉。

整体风格参考：日式禅意 + 北欧简约的结合，
不要有多余的装饰，每个元素都有它存在的理由。

页面大概是这样的：
- 顶部有标题和今天的日期
- 下面有两个小卡片，显示"总共多少"和"完成了多少"
- 有一排标签可以切换看"全部/待办/已完成"
- 中间是任务列表，每个任务可以打勾完成
- 右下角有个很显眼的添加按钮
- 点添加按钮会从底部弹出一个表单
- 底部还有空状态、Toast 提示等

不要用那些烂大街的 UI 库，
我希望每个细节都是我之前没见过的那种，
让人眼前一亮但不张扬。

手机优先，就是那种拿起来就能用、不用学就会的感觉。
```

---

## 第二步：让它可以添加任务

```
现在需要让它可以真的用起来：

点右下角的添加按钮，可以弹出那个表单，
填一下任务名称（可选的描述、优先级），点提交，
任务就出现在列表里了。

数据要存在本地，刷新页面不丢失。
每次添加、完成、删除任务时，给个轻轻的提示。

交互要自然，不要有任何突兀的感觉。
比如按钮按下要有反馈，弹窗出来要顺滑，
任务添加或完成时要有动画，让整个过程看起来是"活"的。

就这些，不用改别的，先把添加这个流程走通。
```

---

## 第三步：让它可以编辑、完成、删除

```
现在让它完整起来：

1. 点任务旁边的按钮可以编辑内容（标题、描述、优先级），
   编辑完保存，列表里直接更新，不用刷新页面

2. 点任务左边的圆形按钮可以标记完成/未完成，
   完成了要有明显的视觉效果（比如划线、变灰），
   整个过程要有动画，不是突然变掉

3. 点删除按钮可以删掉一个任务，
   删之前弹个确认，毕竟删东西要谨慎
   删除也有动画，看起来是"滑走"的感觉

4. 顶部的筛选标签要真的能用，
   切换的时候列表要平滑过渡

```

---

## 使用方式

| 步骤 | 操作 |
|------|------|
| 第一步 | 把"第一步"的描述发给 AI，让它出页面 |
| 第二步 | 把"第二步"的描述发给 AI，让它加功能 |
| 第三步 | 把"第三步"的描述发给 AI，让它补完 |

每一步之间可以跟 AI 说"改一下 XXX"来微调，直到满意为止。

Vibe Coding 的关键是：**先有个大概的感觉，不满意就说，满意了就 next step**。
同时，每一步的描述要尽可能详尽，完整。每一次的改动不要太大，要小步迭代，步子迈得太大容易失控。
不要一开始就追求完美，先跑起来再说。
