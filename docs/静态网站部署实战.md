# 静态网站部署实战

> 把你的网页分享给任何人，让他们通过一个链接就能访问。

---

## 一、什么是"部署"

### 本地 vs 线上

到目前为止，我们都是直接双击 HTML 文件在浏览器打开。这是**本地访问**：

```
你的电脑 ──双击──▶ HTML 文件 ──▶ 浏览器显示
```

**部署**的意思是：把 HTML 文件上传到一台"云端电脑"（服务器），让任何人都可以通过网址访问：

```
任何人 ──访问网址──▶ 云端服务器 ──▶ HTML 文件 ──▶ 浏览器显示
```

### 静态网站的特点

我们的 TodoList 属于**静态网站**：

| 类型 | 说明 | 例子 |
|------|------|------|
| 静态网站 | 文件固定不变，没有数据库 | TodoList、图片站、博客 |
| 动态网站 | 服务器实时生成内容，有数据库 | 微博、淘宝、后台系统 |

静态网站的优势：**简单、快速、不需要服务器维护**。

## 二、部署平台选择

适合初学者的免费部署平台：

| 平台 | 优点 | 缺点 |
|------|------|------|
| **Netlify** | 拖拽即部署，无需命令行 | 需要邮箱注册 |
| **Vercel** | 部署极快，与 GitHub 深度集成 | 需要邮箱注册 |
| **GitHub Pages** | 完全免费，与代码仓库绑定 | 需要 Git 操作知识 |
| **Surge.sh** | 命令最简单，一行搞定 | 需要邮箱注册 |

**本教程推荐 Netlify**，因为它的网页拖拽部署最符合初学者直觉。

---

## 三、Netlify 部署（网页拖拽方式）

这是最简单的方式，**不需要安装任何软件**，只需要一个邮箱。

### 第一步：注册账号

1. 打开 https://app.netlify.com
2. 点击 "Sign up"
3. 用 **邮箱注册**（或用 GitHub 账号登录）
4. 打开邮箱，点击验证链接

### 第二步：准备你的网站文件

确保你的文件夹结构是这样的：

```
my-website/
  └── index.html    ← 必须有一个 index.html
  └── 其他文件...
```

### 第三步：拖拽上传

1. 打开 https://app.netlify.com/drop
2. 把你的文件夹**拖拽**到页面中央的虚线区域
3. 等待几秒，上传完成

### 第四步：访问你的网站

上传成功后，Netlify 会给你一个临时网址，格式类似：

```
https://xxxxx-xxxxx.netlify.app
```

点击这个链接，任何人都能访问你的网站了！

### 第五步（可选）：自定义域名

如果觉得默认域名太长，可以绑定自己的域名：

1. 点击 Netlify 控制台里的 **Domain settings**
2. 在 **Custom domains** 里输入你的域名
3. 按照提示在你的域名服务商处配置 DNS

## 四、Netlify 部署（命令行方式）

命令行部署适合需要频繁更新的项目，比如每次改完代码一键重新部署。

### 第一步：安装 Node.js

Netlify CLI 需要 Node.js 环境。

1. 打开 https://nodejs.org
2. 下载 LTS（长期支持版）
3. 双击安装包，一路下一步

安装完成后，打开终端，输入：

```bash
node --version
npm --version
```

能看到版本号说明安装成功。

### 第二步：安装 Netlify CLI

在终端里运行：

```bash
npm install -g netlify-cli
```

等待安装完成。

### 第三步：登录 Netlify

```bash
netlify login
```

这会打开浏览器，让你授权。点击 "Authorize" 即可。

### 第四步：部署

进入你的网站文件夹，运行：

```bash
cd /path/to/my-website    # 进入文件夹
netlify deploy --dir . --prod    # 部署到生产环境
```

如果是第一次部署，CLI 会问你要不要创建一个新站点，输入 `y` 同意即可。

### 第五步：更新网站

修改代码后，重新运行：

```bash
netlify deploy --dir . --prod
```

即可更新线上版本。

## 五、Vercel 部署（命令行方式）

Vercel 是另一个很受欢迎的平台，部署速度极快。

### 第一步：安装

```bash
npm install -g vercel
```

### 第二步：登录并部署

```bash
cd /path/to/my-website    # 进入文件夹
vercel                    # 第一次部署（预览）
vercel --prod             # 发布到生产环境
```

同样是打开浏览器授权，然后自动部署。

### 特点

- 不需要创建项目，CLI 会自动识别
- 预览模式和正式模式分离
- 国内访问速度较快

## 六、GitHub Pages 部署（适合有 Git 基础的同学）

如果你会用 Git，这个方法是完全免费的，而且可以绑定自定义域名。

### 第一步：创建 GitHub 仓库

1. 打开 https://github.com
2. 登录后，点击右上角 **+** → **New repository**
3. 填写仓库名，如 `my-todolist`
4. 选择 **Public**
5. 点击 **Create repository**

### 第二步：上传文件

在仓库页面，点击 **uploading an existing file**，把网站文件夹里的所有文件拖进去，点击 **Commit changes**。

### 第三步：启用 GitHub Pages

1. 进入仓库的 **Settings**
2. 左侧菜单找到 **Pages**
3. **Source** 选择 `Deploy from a branch`
4. **Branch** 选择 `main`，文件夹选 `/ (root)`
5. 点击 **Save**

### 第四步：访问你的网站

等待 1-2 分钟，刷新页面，顶部会显示你的网站地址：

```
https://你的用户名.github.io/仓库名/
```

## 七、常见问题

### Q：网站能打开，但 CSS 样式丢失了？

**原因**：文件路径问题。检查 HTML 里引用 CSS 的路径是否正确。

**解决**：

```html
<!-- 正确：相对路径 -->
<link rel="stylesheet" href="style.css">

<!-- 错误：绝对路径（本地文件才能用） -->
<link rel="stylesheet" href="/Users/xxx/style.css">
```

### Q：部署后打开是空白页？

**原因**：可能有 JavaScript 报错，或者 HTML 文件名不是 `index.html`。

**解决**：
1. 浏览器按 F12，打开开发者工具
2. 切换到 Console 标签
3. 查看红色错误信息

### Q：修改代码后刷新网站没变化？

**原因**：浏览器缓存。

**解决**：
1. Windows：Ctrl + Shift + R
2. Mac：Cmd + Shift + R
3. 或者清空浏览器缓存再刷新

### Q：部署后图片/字体不显示？

**原因**：这些资源的路径是绝对路径，线上访问不到。

**解决**：把路径改成相对路径。

### Q：我想换一个新的子域名怎么办？

在 Netlify 控制台里：
1. 进入 **Site settings** → **Change site name**
2. 输入新的名字
3. 确认后你的网站就会用新的域名

## 九、各平台对比速查

| 场景 | 推荐平台 | 命令 |
|------|---------|------|
| 第一次部署，想简单 | Netlify 拖拽 | 无命令行，纯网页操作 |
| 经常更新代码 | Netlify CLI / Vercel | `netlify deploy --prod` |
| 有 Git 基础 | GitHub Pages | git push 后自动部署 |
| 国内访问为主 | Vercel / 腾讯云 COS | 同上 |
